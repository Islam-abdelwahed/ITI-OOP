C++ DBMS Engine

SQL Server Inspired Architecture

Created by the Development Team

1. Architecture: SQL Server Style

Event-Driven Logic with Physical File Mapping.

The Event

User inputs a SQL Query
(e.g., INSERT, SELECT).

➜
The Structure

Like SQL Server, Logical Tables map to Physical Files on Disk.

➜
Execution

Logic processes data in RAM & updates files.

2. Storage: CSV Standard

  • Adopting the CSV (Comma Separated Values) standard.
  • One Table = One .csv File (Modular Design).
  • Persistence: Mimics SQL Server's physical storage model.
  • Direct compatibility with Excel & Data Tools.
Users.csv
ID,Name,Age   // Header
1,Ahmed,25      // Record 1
2,Sarah,30      // Record 2
3,Ali,22        // Record 3

3. Workflow: I/O Operations

Bridging the gap between Storage (HDD) and Memory (RAM).

1
Loading Phase (ifstream)
Parsing .csv file into vector<vector<string>>.
2
Processing Phase (RAM)
Performing Search, Filter, and Edit operations in Memory.
3
Commit Phase (ofstream)
Rewriting the .csv file to save changes permanently.
